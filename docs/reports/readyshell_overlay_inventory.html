Observations

  • Overlay 2 is effectively full at 13869 bytes of 14336 (96.7%).
  • Overlay 1 is also large at 13005 bytes (90.7%).
  • The resident heap below the overlay load address is only 2346 bytes, so large working sets depend on overlays and REU-backed storage.
  • Command overlays 3-6 are smaller, but they pay a disk-load cost on every command invocation because they are not cached in fixed REU banks today.
  • Overlay 2 owns the shared formatting buffers, which is why it consumes almost the entire overlay window.
