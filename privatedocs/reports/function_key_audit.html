Conflict Pressure by Key

F1

Still hard to normalize
  • Copy in editor, quicknotes, task list, and calendar 26.
  • New card in dizzy.
  • Format menu in simple cells.
  • Load all in launcher.

F3

Still hard to normalize
  • Paste in editor-family apps, calc plus, and calendar 26.
  • Edit card in dizzy.
  • Drive toggle/select in simple files and multiple dialogs.
  • Formula edit in simple cells.
  • Load selected app in launcher.

F5

Healthier now
  • Mostly save/export or refresh.
  • Clipboard now reinforces that pattern instead of fighting it.
  • Low enough pressure that local variation is acceptable.

F6

Mode-heavy
  • Select in editor and quicknotes.
  • Angle mode in calc plus.
  • Save as in task list.
  • Mode/view toggles in hex viewer and dizzy.

F7

Open/load/view mix
  • Open in editor, quicknotes, task list, and simple cells.
  • Load/import in clipboard.
  • Next view in calendar 26.
  • Mode/focus work in calc plus and dizzy.

F8

Best current convention
  • Every app with a dedicated help key now lands here.
  • Remaining gap is only the apps with no help surface yet.
  • This is the strongest convention to preserve and expand.

Other Hotkeys

System-Level Conventions

  • F2/F4: app switching
  • CTRL+B: launcher/home
  • RUN/STOP: often quit, cancel, or close
  • RETURN: usually open, edit, confirm, launch, or close help

Control-Key Clusters

  • Editor: CTRL+A save as, CTRL+E line end, CTRL+F/G find, CTRL+N/P page
  • Quicknotes: CTRL+L/O pane, CTRL+N/P page, CTRL+W new note, CTRL+R title, CTRL+D delete, CTRL+U/K move, CTRL+F/G find
  • Simple Files: CTRL+N/P page

Letter-Key Workflows

  • Simple Cells: C/V copy/paste
  • Simple Files: C copy, N copy as, D duplicate, S swap panes
  • Calendar 26: L load, R rebuild, N new, A edit, D delete, T set today

More Letter-Key Workflows

  • Dizzy: L load, R rebuild, D/A/S toggle flags, / search, C clear search
  • Deminer: B/I difficulty, P pause, R restart, M menu, J/RETURN reveal, K/SPACE flag
  • Ready Shell: typed HELP, VER, CLEAR

Suggestions

What is easy right now

  • Add F8 help to clipboard, REU viewer, read.me, and 2048.
  • Keep reinforcing the bias of F5 toward save/refresh.
  • Keep reinforcing the bias of F7 toward open/load/navigation.
Important context:
There is no established user base yet, so this is the cheapest moment to normalize conventions. The limiting factor is implementation effort, not user retraining or habit breakage.

What is still harder

  • A full semantic cleanup of F1.
  • A full semantic cleanup of F3.
  • A fully uniform meaning for F6 and F7.

Why it is harder

  • Those keys already carry meaningful editing and navigation flows.
  • Several apps use them deeply enough that changes would affect core interaction, not just labels.

Current best convention

  • F8 = Help
  • F2/F4 app switching
  • CTRL+B launcher/home
